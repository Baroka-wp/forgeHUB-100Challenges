# Chapitre 3 : Le Produit Minimum Viable (MVP)

> "Si vous n'êtes pas embarrassé par la première version de votre produit, c'est que vous l'avez sorti trop tard." — **Reid Hoffman**

### Construire Moins pour Apprendre Plus
Le but d'un MVP n'est pas de construire une version "cheap" de votre produit, mais de construire l'unité minimale nécessaire pour tester vos hypothèses les plus risquées.

<div class="callout">
  <h4>L'Unité Testable</h4>
  Un MVP peut être une simple vidéo (Dropbox), une page de vente (Buffer), ou même un service manuel. Il doit seulement démontrer au client ce qu'il va obtenir.
</div>

### Le Concierge MVP
Pour mieux connaître vos clients, évitez l'automatisation au début. Donnez à chaque nouveau client le "traitement concierge". Faites tout manuellement. Cela vous permet de comprendre les nuances du besoin avant de figer le processus dans du code.

> "N'importe quel produit qui a besoin d'un manuel pour fonctionner est cassé." — **Elon Musk**

### Le Positionnement : La Bataille pour votre Esprit
Al Ries et Jack Trout expliquent que le positionnement est un outil de communication pour atteindre les clients dans un marché encombré.

*   **Si vous êtes le leader** : Rappelez que vous êtes "l'original" (Coca-Cola).
*   **Si vous êtes un challenger** : Trouvez une position inoccupée ou repositionnez le leader (Avis : "Nous sommes n°2, donc nous essayons plus fort").

<div class="callout">
  <h4>Le Framework Canva : Canaux</h4>
  Comment allez-vous atteindre vos clients ? Ne essayez pas d'être partout. Choisissez un ou deux canaux où vos adopteurs précoces sont déjà présents et concentrez vos efforts.
</div>

### Éviter le "Feature Creep"
L'ajout constant de fonctionnalités est une forme de gaspillage. Chaque effort qui ne contribue pas directement à l'apprentissage est une perte de ressources.
